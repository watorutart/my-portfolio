# ADR-0001: ブログの Web フレームワークに Yesod を採用する

## ステータス

承認済み (2026-06-27)

## コンテキスト

Haskell で個人ブログを構築するにあたり、技術選定を行う必要がある。
主な候補は以下の通り。

- **Hakyll** — Haskell 製の静的サイトジェネレーター。Pandoc ベースで、GitHub Pages 等に無料デプロイ可能。Haskell ブログのデファクト。
- **Yesod** — 型安全を徹底した Haskell Web フレームワーク。ルーティング・テンプレート・DB 層すべてがコンパイル時にチェックされる。
- **Servant** — 型レベル API 定義が特徴のフレームワーク。API サーバー向き。
- **Scotty** — 軽量な WAI ベースフレームワーク。Sinatra ライク。

## 決定

**Yesod** を採用する。DB は開発時に **SQLite**、本番では必要に応じて **PostgreSQL** への移行を検討する。

## 理由

- 本プロジェクトの主目的は **Haskell での Web 開発を深く理解すること** であり、単にブログを公開することではない
- Yesod は型安全なルーティング、Persistent (ORM)、Shakespeare テンプレートなど、Haskell らしい Web 開発の概念を幅広くカバーしている
- 動的機能（管理画面、記事 CRUD、タグ検索など）を自前で実装することで、フレームワークの理解が深まる
- 成熟したフレームワークであり、書籍やドキュメントが充実している

## トレードオフ

- Hakyll と比べてデプロイの手間・コストが増える（サーバープロセスの運用が必要）
- Template Haskell や型レベルプログラミングなど、学習コストが高い
- 静的ブログで十分な要件に対してはオーバーエンジニアリングである

## 補足

ビルドツールには **Stack** を使用する（Yesod の scaffolding が Stack 前提のため）。

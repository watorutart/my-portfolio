# モックアップ B: タブ切り替え型ポートフォリオ

TDD サイクル（テストリスト → Red → Green → Refactor）をタブで表現する構成。
初期表示は Green タブ（完成品を最初に見せる）。
Red タブは常に1つだけ（TDD の「テストは1つずつ」を体現）。

│  [ロゴ]  Profile   Blog                 [■]  │  ← ヘッダー (ナビ)
├──────────────────────────────────────────────┤
│                                              │
│          Red → Green → Refactor              │  ← キャッチコピー
│                                              │
│    TDD でものづくりしています。                 │
│                                              │
├──────────────────────────────────────────────┤
│                                              │
│  📋 テストリスト  🔴 Red  [🟢 Green]  🔵 Refactor │  ← タブ（Green が初期選択）
│  ─────────────────────────────────────────   │
│                                              │

## タブごとの内容

### 📋 テストリスト タブ
- 構想中・やりたいプロジェクトの一覧
- 「期待する振る舞いをリストアップする」
- 複数並んでOK（これからやることの洗い出し）

### 🔴 Red タブ
- 今まさに取り組んでいる **1つだけ** のプロジェクト
- 「テストリストから1つ選び、失敗するテストを書く」
- 常に1つしか表示しない（TDD の原則）

### 🟢 Green タブ（初期表示）
- 完成したプロジェクト
- 「テストが通る最小限のコードを書く」
- 複数並ぶ

### 🔵 Refactor タブ
- 改善・リファクタリング済みのプロジェクト
- 「動くコードをより良くする」
- 複数並ぶ

## デザインルール

- リンク色: 未訪問=赤、訪問済み=緑、更新あり=青
- プロジェクトカード: 左上にステータスカラー
- タブの並び順は テストリスト → Red → Green → Refactor（TDD サイクル順）
- 初期選択は Green（完成品を最初に見せる）
- Red タブは常に1つだけ表示（TDD の「テストは1つずつ」を体現）

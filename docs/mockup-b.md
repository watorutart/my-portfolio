# モックアップ B: タブ切り替え型ポートフォリオ

TDD サイクル（Red → Green → Refactor）をタブで表現する構成。
初期表示は Green タブ（完成品を最初に見せる）。

```
┌──────────────────────────────────────────────┐
│  [ロゴ]  Profile   Blog                 [■]  │  ← ヘッダー (ナビ)
├──────────────────────────────────────────────┤
│                                              │
│          Red → Green → Refactor              │  ← キャッチコピー
│                                              │
│    TDD でものづくりしています。                 │
│                                              │
├──────────────────────────────────────────────┤
│                                              │
│    🔴 Red    [🟢 Green]    🔵 Refactor       │  ← タブ（Green が初期選択）
│  ─────────────────────────────────────────   │
│                                              │
│  ┌───────────┐ ┌───────────┐ ┌───────────┐  │
│  │ 🟢        │ │ 🟢        │ │ 🟢        │  │
│  │ tdd-calc  │ │ cli-tool  │ │ chat-app  │  │
│  │           │ │           │ │           │  │
│  │ 完成      │ │ 完成      │ │ 完成      │  │
│  │ Haskell   │ │ Go        │ │ TypeScript│  │
│  └───────────┘ └───────────┘ └───────────┘  │
│                                              │
├──────────────────────────────────────────────┤
│  © 2026 TDD野郎.                              │
└──────────────────────────────────────────────┘
```

## タブごとの内容

### 🔴 Red タブ
- 進行中・構想中のプロジェクト
- 「まず失敗するテストを書く」

### 🟢 Green タブ（初期表示）
- 完成したプロジェクト
- 「テストが通る最小限のコードを書く」

### 🔵 Refactor タブ
- 改善・リファクタリング済みのプロジェクト
- 「動くコードをより良くする」

## デザインルール

- リンク色: 未訪問=赤、訪問済み=緑、更新あり=青
- プロジェクトカード: 左上にステータスカラー
- タブの並び順は Red → Green → Refactor（TDD サイクル順）
- 初期選択は Green（完成品を最初に見せる）
